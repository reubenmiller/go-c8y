# Contributing

## Development workflow

```bash
task test-ci        # offline gate: unit tests + offline integration suite + coverage
task test-offline   # just the offline integration suite (no Cumulocity tenant needed)
task lint-api       # report drift between the OpenAPI spec and the SDK
task generate       # regenerate the spec-derived code (see below)
```

The offline suite runs against an in-memory fake server and needs no live tenant.
See [docs/OFFLINE_TESTING.md](docs/OFFLINE_TESTING.md).

## Generated code — edit the spec/overlay, not `zz_generated_*.go`

Part of the SDK is **generated from the Cumulocity OpenAPI spec** by
[`tools/c8ygen`](tools/c8ygen). Generated files are named `zz_generated_*.go` and carry a
`// Code generated by tools/c8ygen; DO NOT EDIT.` header. The design and rationale are in
[docs/API_GEN.md](docs/API_GEN.md).

There are three inputs you may edit; the generated files are **outputs** and must never
be hand-edited:

| Input | What it is |
|---|---|
| [`docs/c8y-oas.yml`](docs/c8y-oas.yml) | The vendored upstream OpenAPI spec. Refresh with `task fetch-spec`. |
| [`docs/c8y-oas.overlay.yml`](docs/c8y-oas.overlay.yml) | SDK-specific codegen directives (option structs, type/doc overrides, models) and the drift-waiver list. Survives `fetch-spec`. |
| The hand-written `*.go` | The ergonomic layer (resolvers, `Upsert`, iterators, nested-object accessors, constructors) that composes the generated substrate. |

After changing any input, run **`task generate`** and commit the regenerated files. CI
fails if `zz_generated_*` is stale (the `api-drift` job runs `task generate` and diffs).

### Adding a resource to the generator

1. Append a `resources:` entry to `docs/c8y-oas.overlay.yml` (see the `alarms`/`events`
   entries for the shape: option struct path/method, field type/doc overrides, embeds,
   and model schema + skipped nested props).
2. Run `task generate`.
3. Delete the now-superseded hand-written option struct and the scalar model accessors;
   keep the resolver field, nested-object accessors (e.g. `SourceID`), and constructors.
4. Run `task test-offline` — the generated surface must be behaviour-identical.

### API drift is gated

`task lint-api -- --strict` (CI's `api-drift` job) fails on any OAS↔SDK drift **not**
declared under `drift:` in the overlay. If you add or remove an endpoint, either implement
the matching SDK path or record the decision in the overlay's `ignoreMissing` /
`ignoreExtra` list with a comment. Known coverage gaps are tracked there too.

## Commits & PRs

- Target the `main` branch.
- Keep generated files in the same commit as the input change that produced them.
- Run `task test-ci` before opening a PR.
