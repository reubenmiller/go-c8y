# Offline Integration Testing

This document describes the offline testing infrastructure for the go-c8y SDK. It enables running the full integration test suite against a stateful in-memory fake Cumulocity server, with no network or live server required.

**Test results:** 378 PASS / 20 SKIP / 0 FAIL in ~13 seconds

---

## Quick Start

```bash
# Run all tests offline (default, no server needed)
task test-offline

# Run against a live Cumulocity server
task test-live

# Record golden files from a live server
task test-record

# Validate fake server against golden files
task test-golden
```

Or directly with `go test`:

```bash
# Offline (default when TEST_MODE is unset)
TEST_MODE=offline go test -timeout 2m ./test/c8y_api_test/...

# Live
TEST_MODE=live go test -timeout 30m ./test/c8y_api_test/...
```

## Architecture

```
┌──────────────────────────────────────────────────┐
│  Test                                             │
│  client := testcore.CreateTestClient(t)           │
│       ↓                                           │
│  Mode A (offline):  httptest.Server               │
│       └─ FakeCumulocity (stateful, in-memory)     │
│  Mode B (live):     real C8Y_HOST                 │
│  Mode C (record):   proxy → real server + save    │
│       ↓                                           │
│  Same test code, same assertions, all 3 modes     │
└──────────────────────────────────────────────────┘
```

### Test Modes

Controlled by the `TEST_MODE` environment variable:

| Mode | Env value | Description |
|---|---|---|
| **Offline** | `offline` (default) | `httptest.Server` running the fake server — in-memory, instant |
| **Live** | `live` | Real Cumulocity server via `C8Y_HOST` / `C8Y_USER` / `C8Y_PASSWORD` |
| **Record** | `record` | Proxies to a real server and captures request/response pairs as golden files |

### Why a Stateful Fake Server

| VCR Cassettes | Stateful Fake |
|---|---|
| Brittle — tied to exact request order | Flexible — any test order works |
| Cannot handle dynamic IDs | Generates IDs like the real server |
| Large fixture files per test | Zero fixture files for offline mode |
| Breaks when API evolves | Can be tested and versioned independently |
| One cassette per test | All tests share one server instance |
| Cannot test CRUD workflows | Full create → read → update → delete works |

## Fake Server

**Package:** `internal/pkg/fakeserver/`

An `httptest.Server`-backed fake Cumulocity platform with in-memory stores, supporting CRUD operations, filtering, pagination, and error responses.

### File Structure

```
internal/pkg/fakeserver/
├── server.go              # New(t) creates server, registers routes, seeds default data
├── store.go               # Thread-safe in-memory Store (map + insertion order)
├── helpers.go             # writeJSON, writeError, readBody, extractID
├── pagination.go          # BuildCollectionResponse with statistics + next links
├── filtering.go           # FilterItems: source, type, status, severity, date, text, etc.
├── recorder.go            # RecordingTransport, golden file save/load, CompareRecords
├── recorder_test.go       # Unit tests for the recording/comparison logic
├── handler_alarm.go       # /alarm/alarms (CRUD + count + bulk update/delete)
├── handler_event.go       # /event/events (CRUD + binary attachments)
├── handler_operation.go   # /devicecontrol/operations + /bulkoperations
├── handler_measurement.go # /measurement/measurements (CRUD + series)
├── handler_inventory.go   # /inventory/managedObjects (CRUD + children + CQL queries)
├── handler_identity.go    # /identity/ (external IDs: create, get, delete)
├── handler_application.go # /application/ (CRUD + versions + subscriptions)
├── handler_binary.go      # /inventory/binaries (multipart upload, download, replace)
├── handler_user.go        # /user/ (users, groups, roles, currentUser, permissions)
├── handler_tenant.go      # /tenant/ (tenants, currentTenant, options, statistics)
└── handler_misc.go        # Retention, audit, features, login, certs, device requests, etc.
```

### Store

All resource types use the same `Store` — a thread-safe map keyed by ID, maintaining insertion order:

| Method | Description |
|---|---|
| `Create(body, selfURL)` | Assigns auto-incremented ID, sets `creationTime`/`lastUpdated`/`self` |
| `CreateWithID(id, body)` | Stores with a caller-provided ID (for seed data) |
| `Get(id)` | Returns document or `(nil, false)` |
| `Update(id, patch)` | JSON merge-patches the existing document, updates `lastUpdated` |
| `Delete(id)` | Removes from store, returns `true` if existed |
| `List()` | Returns all documents in insertion order |
| `Count()` | Returns number of stored documents |

IDs are generated by a global atomic counter starting at `10000`.

### HTTP Method Behaviour

| Method | Behaviour | Status |
|---|---|---|
| `POST /resource` | Parse body → assign ID + timestamps + self link → store | `201 Created` |
| `GET /resource` | Apply query-param filters → paginate → return collection | `200 OK` |
| `GET /resource/{id}` | Lookup by ID → return single document | `200 OK` / `404` |
| `PUT /resource/{id}` | JSON merge-patch → update timestamps → return | `200 OK` / `404` |
| `DELETE /resource/{id}` | Remove from store → return empty body | `204 No Content` / `404` |

### Collection Response Envelope

Every list endpoint returns the standard Cumulocity collection format:

```json
{
  "<collectionProperty>": [ ... ],
  "self": "http://server/alarm/alarms?pageSize=5&currentPage=1",
  "statistics": {
    "currentPage": 1,
    "pageSize": 5,
    "totalPages": 2,
    "totalElements": 8
  },
  "next": "http://server/alarm/alarms?pageSize=5&currentPage=2"
}
```

### Query Parameter Filtering

Supported filters (applied as AND logic):

| Parameter | Behaviour |
|---|---|
| `pageSize`, `currentPage` | Pagination (default pageSize=5, currentPage=1) |
| `source` | Match `source.id` |
| `type` | Comma-separated list match |
| `status` | Comma-separated list match |
| `severity` | Comma-separated list match |
| `dateFrom`, `dateTo` | RFC3339 date range filter (truncated to second precision) |
| `fragmentType` | Check if field exists in document (`has()`) |
| `text` | Case-insensitive substring match |
| `name` | Exact name match |
| `query` / `q` | CQL query: `has(fragment)`, `field eq 'value'`, `and` |
| `ids` | Comma-separated list of IDs |

### Seed Data

The fake server seeds baseline data that every Cumulocity tenant has:

- **Tenant:** `t12345` (ACTIVE, domain `t12345.cumulocity.com`)
- **User:** `admin` with ROLE_TENANT_ADMIN
- **Login options:** BASIC (PASSWORD grant, INTERNAL user management)
- **Groups:** admins, devices
- **Retention rule:** 90-day, all types
- **System options:** version 1016.0.0, timezone config, 2FA strategy
- **Tenant options:** access.control, 2FA enforcement
- **Applications:** cockpit, devicemanagement, administration, reporting + 11 microservice apps
- **Managed objects:** TestDevice001 (c8y_Linux) + 3 thin-edge.io devices with c8y_Agent
- **Alarms:** 2 active alarms (WARNING, MAJOR)
- **Trusted certificate:** CA certificate (auto-registration disabled)

### Authentication

The server validates that an `Authorization` header is present (returns 401 if missing) but does not verify actual credentials. Public endpoints (`/tenant/loginOptions`, `/tenant/oauth`) bypass the check.

## Test Client Factory

All tests use `testcore.CreateTestClient(t)` which automatically selects the backend:

```go
// Offline mode: starts fake server, sets env vars
// Live mode: reads C8Y_HOST/C8Y_USER/C8Y_PASSWORD from environment
// Record mode: proxies to live server with RecordingTransport
client := testcore.CreateTestClient(t)
```

Three factory functions are available:

| Function | Description |
|---|---|
| `CreateTestClient(t)` | Standard authenticated client |
| `CreateTestClientNoAuth(t)` | Client without credentials (for 401 tests) |
| `CreateTestClientWithToken(t)` | Client using bearer token auth |

### Test Helpers

| Helper | Description |
|---|---|
| `SkipOffline(t, reason)` | Skip test when `TEST_MODE=offline` |
| `SleepIfLive(d)` | Only sleep when running against a live server |
| `IsOffline()` | Returns true when `TEST_MODE` is `offline` or unset |
| `CreateDevice(t, client)` | Creates a managed object with `c8y_IsDevice` fragment |
| `CreateDeviceAgent(t, client)` | Creates a device with `com_cumulocity_model_Agent` |
| `CreateManagedObject(t, client, payload)` | Creates an arbitrary managed object |

## Golden File Validation

Golden files capture live server request/response pairs and validate that the fake server produces equivalent responses.

### Workflow

```bash
# 1. Record against a live server
TEST_MODE=record go test -count=1 -v ./test/c8y_api_test/...
# → Saves to testdata/golden/<TestName>.json

# 2. Validate fake server matches
TEST_MODE=offline go test -count=1 -v -run TestFakeServer_MatchesGolden ./test/c8y_api_test/...
```

### How Comparison Works

- Requests are replayed against the fake server via `FakeServer.Replay()`
- Matched by `(Method, Path)`
- Checks: status codes match, top-level JSON keys match
- Missing keys in fake response = failure; extra keys = OK (informational)
- Results written as `<TestName>.comparison.json`

Golden files are gitignored (`testdata/golden/`) since they contain live server data.

## Design Decisions

1. **Single interception point** — `testcore.CreateTestClient(t)` switches the backend. No SDK code changes needed.
2. **Stateful stores** — Each test gets its own fake server instance via `fakeserver.New(t)` with fresh stores, preventing cross-test interference.
3. **Real collection envelopes** — Pagination statistics, self links, and next links match the real Cumulocity API format so the SDK's collection parsing works unmodified.
4. **Date precision normalization** — Query parameters use RFC3339 (second precision) while stored documents use RFC3339Nano. The filter layer truncates both to seconds when comparing.
5. **Thread safety** — All stores use `sync.RWMutex` so tests can safely use `t.Parallel()`.
6. **No SDK changes** — The fake server is purely test infrastructure in `internal/`. The SDK code under `pkg/` is untouched.
