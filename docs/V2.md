# Cumulocity Go SDK v2

> **See [API_DESIGN.md](./API_DESIGN.md) for comprehensive API design documentation.**

This v2 rewrite of the go-c8y SDK provides a modern, type-safe, and flexible Go client for Cumulocity IoT. The design emphasizes developer experience, type safety through generics, and rich result metadata.

## Key Features

### Rich Result Type
Every operation returns a `Result<T>` with comprehensive metadata including status, HTTP details, retry information, duration, and request inspection capabilities.

### Flexible JSON Handling
JSONDoc models provide both structured method access and raw JSON access, allowing custom fields and unmarshaling to user-defined types.

### Context-Driven Behavior
Control execution through context:
- **Dry Run** - Prepare requests without sending
- **Deferred Execution** - Resolve parameters, then execute after confirmation
- **Header Redaction** - Control sensitive data visibility in logs

### Smart Resource Resolution
Consistent resolver patterns across services:
- Direct IDs: `"12345"`
- Name lookup: `"name:deviceName"`
- External IDs: `"ext:c8y_Serial:ABC123"`
- Query-based: `"query:type eq 'c8y_Device'"`

### Powerful Pagination
- Collection results with metadata
- Iterator pattern for all items
- Custom unmarshaling during iteration

## Quick Example

```go
package main

import (
    "context"
    "fmt"
    "github.com/reubenmiller/go-c8y/pkg/c8y/c8y_api"
    "github.com/reubenmiller/go-c8y/pkg/c8y/c8y_api/operations"
)

func main() {
    // Create client
    client := c8y_api.NewClient(baseURL, username, password)
    ctx := context.Background()
    
    // List operations with device name resolution
    result := client.Operations.List(ctx, operations.ListOptions{
        DeviceID: "name:my-device",  // Automatically resolved to ID
        Status:   "PENDING",
    })
    
    // Handle result
    if result.Err != nil {
        panic(result.Err)
    }
    
    // Iterate over operations
    for _, op := range result.Data.Items() {
        fmt.Printf("%s: %s\n", op.ID(), op.Status())
    }
}
```

## Migration from v1

Key differences:
1. **Result type** instead of `(data, *Response, error)` tuples
2. **Service names** are now plural (`Operations` not `Operation`)
3. **Options structs** instead of variadic functional parameters
4. **JSONDoc models** with method accessors instead of structs

See [API_DESIGN.md](./API_DESIGN.md#migration-from-v1) for detailed migration guide.
